## Décisions techniques
- **face-api.js plutôt que FaceDetector natif** — Shape Detection API requiert `chrome://flags/#enable-experimental-web-platform-features`, inaccessible par défaut dans Chrome moderne
- **Modèles en local (`public/`) plutôt que CDN** — le package npm de face-api.js n'inclut pas les poids ; CDN `npm` → 404 ; GitHub raw → OK mais dépendance réseau ; `public/` → fiable et offline
- **`document.baseURI` pour l'URL des modèles** — gère automatiquement le sous-chemin GitHub Pages sans configuration supplémentaire
- **Boucle canvas via `setTimeout(0)` dans l'`effect()`** — évite la violation RAF 192 ms causée par le démarrage de la boucle dans le cycle Angular
- **Body transparent en steampunk + fallback sur `:root`** — permet au `z-index: -1` du fond cuivre de passer à travers sans casser les autres thèmes

## Problèmes rencontrés
- 404 sur les modèles face-api.js (CDN npm → GitHub raw → assets locaux : 3 itérations)
- Angular 17+ utilise `public/` et non `src/assets/` pour les fichiers statiques
- Violation RAF 192 ms au premier démarrage du tracking (corrigé)
- `FaceDetector` natif non fonctionnel sans flag Chrome (migré vers face-api.js)
